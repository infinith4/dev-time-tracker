# Claude Code ベストプラクティス

Anthropic公式ベストプラクティスに基づくガイドです。

## 5つの設計思想

### 1. コンテキストウィンドウ管理が基盤

コンテキストは「有限で劣化する」リソース。

**対策:**
- タスク間で `/clear` を使用
- `/compact <指示>` で詳細制御
- 無関係な情報の蓄積を防ぐ

### 2. 検証可能性を優先

テスト実行、スクリーンショット検証、リンターチェックを提供。

**ポイント:**
- Claude自身が作業をチェックできる仕組みを提供
- 検証がない場合、「正しそうだが動作しない」コードが生まれる

### 3. 段階的アプローチ

```
探索 (Explore) → 計画 (Plan) → 実装 (Implement) → 検証 (Verify)
```

- 小規模タスク（タイポ修正など）: 計画をスキップ可
- 複雑な変更: 事前計画が有効

### 4. 環境カスタマイズの体系

| 種類 | 用途 | 特徴 |
|------|------|------|
| CLAUDE.md | 会話開始時に読む設定 | 助言的（コードスタイル好みなど） |
| フック | 特定タイミングでスクリプト実行 | 決定論的（自動フォーマットなど） |
| スキル | 再利用可能なワークフロー | `/skill名` で呼び出し |
| サブエージェント | 別コンテキストで探索 | 読み取り中心タスクに最適 |

### 5. 失敗パターンの認識と回避

| パターン | 対策 |
|---------|------|
| 無関係情報の蓄積 | タスク間で `/clear` |
| 失敗アプローチでの汚染 | 2回失敗後に `/clear` し再出発 |
| 過剰なCLAUDE.md | 削除可能な行を容赦なく削除 |
| 検証ギャップ | テスト・スクリーンショット常備 |
| 無限探索 | スコープ制限またはサブエージェント使用 |

## 7つのベストプラクティス

### 1. 検証方法の提供

最も効き目の高い施策。

```bash
# テスト実行
npm test
pytest tests/
dotnet test
mvn test

# リント
npm run lint
ruff check .
```

### 2. 探索→計画→実装の分離

- **Plan Mode**: 調査し、ファイル読み込みのみ実施
- **Normal Mode**: 計画に沿ってコーディング

複数ファイル変更時に特に有効。

### 3. 具体的なコンテキスト提供

- ファイル参照（@記号）
- 画像ドラッグ&ドロップ
- URLリンク
- パイプでのデータ投入

曖昧さを排除する。

### 4. CLAUDE.md設定

記載すべき内容:
- Claudeが推測できないBashコマンド
- 非デフォルトのルール
- プロジェクト固有のワークフロー

記載不要:
- 言語標準の慣習
- 一般的なコーディングルール

### 5. 効果的な会話

先輩開発者に聞くように質問する。
AskUserQuestionツールでインタビューさせることも可能。

### 6. セッション管理

- 早期の軌道修正が重要
- `/clear` で無関係なタスク間をリセット
- サブエージェントで調査を委任し、メインコンテキストを保持

### 7. 自動化と生産性向上

```bash
# ヘッドレスモード
claude -p "プロンプト"

# パイプ入力
cat file.txt | claude -p "このファイルを分析して"
```

## ディレクトリ構造

```
.claude/
├── settings.json      # 権限設定
├── hooks.json         # フック設定
├── BEST_PRACTICES.md  # このファイル
└── skills/            # スキルファイル
    ├── skill-name/
    │   └── SKILL.md   # スキル定義（新形式）
    └── ...
```

## スキルファイル形式

### 新形式（推奨）

```
.claude/skills/review/SKILL.md
```

フロントマター:
```yaml
---
name: review
description: コードレビューを実行
---
```

### 自動呼び出しの制御

```yaml
---
name: review
description: コードレビューを実行
disable-model-invocation: true  # 自動呼び出しを防ぐ
---
```

## コマンド一覧

| コマンド | 用途 |
|---------|------|
| `/clear` | コンテキストをクリア |
| `/compact <指示>` | コンテキストを要約 |
| `/init` | CLAUDE.mdを生成 |
| `/permissions` | 権限設定 |
| `/hooks` | フック設定 |
| `/skill名` | スキルを呼び出し |
